# YeaHub Test Automation

Фреймворк для автоматизированного тестирования REST API сервиса YeaHub.

## Описание

Проект содержит автоматизированные тесты для проверки функциональности API, включая:
- Регистрацию и аутентификацию пользователей
- Управление профилем
- Сброс и смену пароля
- Верификацию email

## Установка

```bash
# Создать и активировать виртуальное окружение
python -m venv .venv
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # Linux/Mac

# Установить зависимости
pip install -r requirements.txt
```
```
#Python version 
Python 3.14.0
```

## Запуск тестов

```bash
# Все тесты
pytest tests/api/

# Конкретный тест
pytest tests/api/test_post_signup_yeahub.py

# С подробным выводом
pytest tests/api/ -v
```

## Mail tests

Для mail-слоя используется реальный IMAP-ящик. Сейчас в проекте как тестовый ящик
используется `mailbox.org`, потому что он поддерживает IMAP и подходит для проверки
`MailClient`.

### Mail credentials

Креды для IMAP хранятся локально в `.env` и читаются через
`resources/mail_creds.py`.

Используемые переменные:
- `MAIL_HOST`
- `MAIL_PORT`
- `MAIL_EMAIL`
- `MAIL_PASSWORD`
- `MAIL_FOLDER`

`resources/mail_creds.py` нужен только как конфигурационный слой, чтобы не хранить
секреты прямо в коде.

На текущий момент интеграционный тест покрывает именно flow письма верификации
(`Verify Your Email`). Если позже появятся другие mail-flow, для них лучше добавлять
отдельные тесты и отдельные описания.

Важно:
- письмо с верификацией от сервиса может попадать в спам или вообще блокироваться на входе;
- поэтому интеграционный тест не запускается автоматически;
- его нужно запускать вручную только когда вы хотите проверить живой mail-flow.
- если письмо со стенда приходит в другой ящик, можно переслать или заново отправить
  оригинальное письмо на рабочий IMAP-ящик, который читает `MailClient`.

Чтобы прогнать живой mail-flow, нужно:
1. Создать/подготовить рабочий mailbox с IMAP-доступом
2. Отправить на него письмо верификации с сервиса
3. Запустить интеграционный тест вручную

Команда для запуска:

```bash
RUN_MAIL_INTEGRATION=1 ./.venv/bin/python -m pytest tests/mail/test_mail_client_integration.py
```

Если нужно прогнать только mail-тесты:

```bash
./.venv/bin/python -m pytest tests/mail
```

## Структура

- `api/` - API клиенты для взаимодействия с сервисом
- `tests/api/` - Автоматизированные тесты
- `mail/` - Слой для работы с IMAP-почтой, поиска verification email и удаления писем
- `tests/mail/` - Unit и integration тесты для mail-слоя
- `constants/` - Константы и конфигурация
- `conftest.py` - Pytest фикстуры

## Использование линтера
- `ruff check .` - Покажет все ошибки
- `ruff check . --fix` - Линтинг + автофикс
- `ruff format . --diff` - Показывает что будет форматировать
- `ruff format .` - Форматирование всего
- `ruff check . --output-format full`  - Детальный отчет

Автоиспользование перед коммитом:
- `pre-commit install` - Автофикс и форматирование автоматически перед коммитом
- Если при попытке коммита он находит и исправляет ошибки, надо снова делать git add .
